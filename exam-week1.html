github 提交约束

  • 提交信息以增加（add）、优化（optimized）、修正（fixed)开头。（冒号空格，然后是提交信息）

  • 所有 git 操作请使用命令行。

  • 逾期交卷，不合格。
  • github 提交乱写，不合格。
  • 代码整洁度占 10%。
  • 平时表现占附加分 10分。（合格者忽略此项）
